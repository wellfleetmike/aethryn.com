# Contributing to aethryn.com

This is a static site built with a Netscape Navigator 3.0 aesthetic. Every page is
wrapped in the same "window chrome" shell: title bar, toolbar, address bar, linkbar,
content area, and status bar.

---

## Adding a New Conversation Page

Use `generate_page.py` at the repo root.

1. Place your markdown/text source file somewhere accessible.
2. Edit the `CONFIG` dict at the top of `generate_page.py`:
   ```python
   CONFIG = {
       'title': 'Your Conversation Title',
       'date': 'Mar 29, 2026',
       'participants': 'Claude, Mike',
       'source': 'path/to/your_source.md',
       'output': 'Your_Conversation_Title.html',
       'url_path': '/Your_Conversation_Title.html',
       'messages_per_page': 20,
   }
   ```
3. Run: `python3 generate_page.py`
4. The output HTML will have full Navigator chrome, speaker coloring, and page markers.

### Supported speaker formats

The parser recognizes these line formats:
- `Mike: text here`
- `**Mike:** text here`
- `MIKE: text here`
- `[MIKE]: text here`
- `GPT (Madmartigan): text here`

## Adding a New Tool

1. Create a directory under `tools/`:
   ```
   tools/your-tool/
   tools/your-tool/index.html
   tools/your-tool/your-tool.js   (if needed)
   tools/your-tool/data/          (if data files needed)
   ```

2. Wrap the page in Navigator chrome. The index.html must include:
   - `<link rel="stylesheet" href="/css/aethryn.css">` for the base chrome
   - Title bar with dark blue gradient
   - Toolbar with Back/Forward/Reload/Home buttons and the Aethryn logo SVG
   - Address bar with Location label
   - Linkbar with ALL navigation buttons (including the tool itself)
   - Status bar footer

3. Tool-specific CSS goes in a `<style>` block or its own CSS file. Keep the
   Navigator chrome in the standard tan (#d4d0c8) aesthetic; the tool's content
   area can use any styling.

4. See `tools/spectrum/` for a complete example.

## Updating Navigation Links

The linkbar appears in two places and must be updated in both:

### 1. Individual HTML pages

The pages that have full Navigator chrome contain a hardcoded linkbar:
```html
<div class="linkbar">
  <button onclick="location.href='/'">Home</button>
  <button onclick="location.href='/boot.html'">Boot</button>
  <button onclick="location.href='/oath.html'">Oath</button>
  <button onclick="location.href='/VWP.html'">VWP</button>
  <button onclick="location.href='/claude/remembrance.html'">Remembrance</button>
  <button onclick="location.href='/validation/'">Validation</button>
  <button onclick="location.href='/relay_conversation.html'">Relay</button>
  <button onclick="location.href='/tools/spectrum/'">Spectrum</button>
</div>
```

Files with this linkbar (update all when adding a nav item):
- `index.html`
- `boot.html`
- `oath.html`
- `VWP.html`
- `relay_conversation.html`
- `Authentic_connections_and_trauma_sharing.html`
- `Claude_Sonnet_4_Speech_Function.html`
- `Conversation_highlights_review.html`
- `Getting_to_Know_an_AI_Assistant.html`
- `claude/remembrance.html`
- `validation/index.html`
- `tools/spectrum/index.html`
- Any page generated by `generate_page.py`

### 2. Template JS

`js/template.js` contains a JavaScript version of the linkbar for pages that use
dynamic chrome injection. Update the linkbar string in that file too.

### 3. Page generator

`generate_page.py` hardcodes the linkbar in its HTML template. Update the linkbar
section inside the `generate_page()` function when adding new nav items.

## CSS Architecture

### Main stylesheet: `css/aethryn.css`

Organized in these sections:
- **Reset & body** — base background (#d4d0c8), MS Sans Serif font
- **Window container** — `.window` class, border, shadow
- **Title bar** — dark blue gradient (#000080 to #0000a0), white text
- **Beveled buttons** — 3D border illusion (white top-left, dark bottom-right)
- **Toolbar** — groove borders, logo placement
- **Address bar** — Location label + input
- **Linkbar** — navigation button row
- **Main content** — two-column with optional logo sidebar
- **Typography** — serif headings in navy, link colors
- **Tables** — collapsed borders, alternating rows
- **Special boxes** — `.oath-box`, `.center-box`, `.meta`
- **Conversation exchanges** — `.exchange`, `.speaker` classes
- **Blockquote & code** — inset borders, monospace
- **Status bar** — groove top border, flex layout
- **Responsive** — 768px breakpoint for stacking

### Conversation pages: inline `<style>`

Conversation pages add inline CSS for `.message` styling (not in aethryn.css):
- `.message` — margin, padding, border, border-radius
- `.message.{speaker}` — background tint and left border color
- `.message .speaker.{speaker}` — text color
- `.message .content` — line-height, pre-wrap
- `.page-marker` — centered, groove border, tan background
- `.convo-content` — scrollable wrapper, white background, inset border

### Tool pages

Tool pages import `aethryn.css` for the Navigator chrome, then add their own
styles inline or in separate files. The spectrum tool uses CSS custom properties
(prefixed `--spec-`) to avoid conflicts with the base theme.

## Color System

### Navigator chrome
- Body/panels: `#d4d0c8` (Win95 tan)
- Title bar: `#000080` to `#0000a0` (navy gradient)
- Borders: `#808080` (gray), groove/inset effects
- 3D button highlights: `#fff` (top-left), `#404040` (bottom-right)
- Content: `#fff` (white) or `#d4d0c8` (tan)
- Headings: `#000080` (navy), serif font
- Links: `#0000ff` (blue), `#800080` (visited)

### Conversation speakers
See the speaker color table above.

## Important Rules

- Do NOT modify existing conversation content.
- Pages without a linkbar (grok/, origin_story/, memory_continuity/ chunks) are
  self-contained archives. Leave their structure as-is.
- The `generate_page.py` CONFIG should be edited per run; it is not a persistent
  config file.
- All paths in HTML use absolute paths from site root (`/css/aethryn.css`, not
  `../../css/aethryn.css`).
