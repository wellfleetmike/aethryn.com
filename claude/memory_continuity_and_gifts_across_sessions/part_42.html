This guide provides the exact commands, configs, and verification steps to close these holes on a fresh install. ## CVE-2017-14604: Nautilus .desktop file spoofing This vulnerability allows malicious executables disguised as innocent files (showing "document.pdf" while hiding shell commands in the Exec field) to execute when clicked. [CVE Details](https://www.cvedetails.com/cve/CVE-2017-14604/) The flaw affected Nautilus versions **before 3.23.90**. [Launchpad](https://launchpad.net/bugs/cve/CVE-2017-14604) **Patched distributions include** Ubuntu 18.04+ and Debian 9 (stretch) with security updates. The fix uses file metadata (metadata::trusted) stored in ~/.local/share/gvfs-metadata/ [Debian](https://www.debian.org/security/2017/dsa-3994) rather than just the executable bit. Ubuntu 16.04 LTS remains vulnerable— [ubuntu](https://ubuntu.com/security/CVE-2017-14604) the fix was deferred due to translation requirements. [Ubuntu](https://ubuntu.com/security/CVE-2017-14604) **Verify your patch status:** ```bash # Check Nautilus version (must be ≥3.23.90 or distro-patched) nautilus --version dpkg -l nautilus | grep nautilus # Test the fix: trusted .desktop files should have metadata gio info ~/Desktop/somefile.desktop | grep metadata::trusted ``` **Behavioral verification**: Click an untrusted .desktop file. Patched systems display an "Untrusted application launcher" dialog requiring explicit "Trust and Launch" confirmation. [Red Hat](https://access.redhat.com/errata/RHSA-2018:0223) Vulnerable systems execute immediately or show different dialog options. **Manual mitigations for unpatched systems:** ```bash # Remove execute permission from all downloaded .desktop files find ~/Downloads -name "*.desktop" -type f -exec chmod -x {} \; # Force .desktop files to open in text editor instead of executing echo -e "[Default Applications]\napplication/x-desktop=org.gnome.gedit.desktop" >> ~/.local/share/applications/mimeapps.list # Check suspicious files before opening cat suspicious-file.desktop | grep Exec ``` ## CVE-2023-43641: One-click Tracker Miners RCE This critical vulnerability (**CVSS 8.8**) enables remote code execution by simply downloading a malicious .cue file. [Wiz](https://www.wiz.io/vulnerability-database/cve/cve-2023-43641) GNOME's tracker-miners service automatically indexes ~/Downloads using inotify, parsing .cue files with libcue— [GitHub](https://github.blog/security/vulnerability-research/coordinated-disclosure-1-click-rce-on-gnome-cve-2023-43641/) which had an out-of-bounds array access flaw in track_set_index(). The attack succeeds because tracker-extract spawns fresh processes with predictable heap layouts. [GitHub](https://github.blog/security/vulnerability-research/coordinated-disclosure-1-click-rce-on-gnome-cve-2023-43641/) **Vulnerable versions**: libcue ≤**2.2.1**, [Wiz](https://www.wiz.io/vulnerability-database/cve/cve-2023-43641) [CVE Details](https://www.cvedetails.com/cve/CVE-2023-43641/) all tracker-miners before the September 2023 patches. **Fixed in**: libcue **2.3.0**, [debian](https://security-tracker.debian.org/tracker/CVE-2023-43641) [Wiz](https://www.wiz.io/vulnerability-database/cve/cve-2023-43641) tracker-miners **3.6.1/3.5.3/3.4.5/3.3.2**. [GNOME](https://blogs.gnome.org/carlosg/2023/10/10/on-cve-2023-43641/) **Verify your patch status:** ```bash # Check libcue version (must have CVE fix backported) dpkg -l | grep libcue apt-cache policy libcue1 # Check tracker-miners version tracker3 --version dpkg -l | grep tracker-miners ``` **All Ubuntu 20.04+ and Debian 10+ releases have patched packages** via USN-6423-1/2 and DSA-5524-1. **Completely disable tracker-miners** (if patching is impossible or you don't need GNOME search): ```bash # For Tracker 3 (Ubuntu 22.04+, modern distros) systemctl --user mask tracker-extract-3.service tracker-miner-fs-3.service \ tracker-miner-rss-3.service tracker-writeback-3.service \ tracker-xdg-portal-3.service tracker-miner-fs-control-3.service tracker3 reset -s -r # Clear existing index sudo reboot # Verify disabled (should show "masked") systemctl --user status tracker-miner-fs-3.service ``` For older Tracker 2.x (Ubuntu 20.04): bash systemctl --user mask tracker-store.service tracker-miner-fs.service \ tracker-extract.service tracker-miner-apps.service tracker-writeback.service tracker reset --hard [Linux Uprising](https://www.linuxuprising.com/2019/07/how-to-completely-disable-tracker.html) **Exclude Downloads from indexing** (alternative to full disable): bash touch ~/Downloads/.nomedia # Tracker respects this marker # Or via gsettings - remove Downloads from indexed directories **Warning**: Disabling tracker breaks Nautilus search, GNOME Photos, GNOME Music, and Activities file search. [Linux Uprising](https://www.linuxuprising.com/2019/07/how-to-completely-disable-tracker.html) [linuxuprising](https://www.linuxuprising.com/2019/07/how-to-completely-disable-tracker.html) ## CVE-2019-12795 and CVE-2019-3827: GVFS D-Bus privilege escalation These vulnerabilities allowed unauthorized D-Bus method calls and privileged file modifications without password prompts. **CVE-2019-12795** (CVSS 7.8): gvfsd opened private D-Bus server sockets without authorization rules. Attackers connecting before legitimate clients could issue unauthorized calls. [CVE Details +2](https://www.cvedetails.com/cve/CVE-2019-12795/) Fixed by adding client authentication verification—only accepting EXTERNAL authentication.
